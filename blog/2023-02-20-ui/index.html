What does lookup_bigtest do, anyways?

lookup_bigtest is an integration test, it creates test data (~50K orders with ~100K order items) and runs Capillaries script that:

  • reads orders from 10 files and order items from 100 files in parallel
  • performs four types of joins in parallel
  • saves sorted results as parquet files

The diagram was generated by this Capillaries command:

          cd pkg/exe/toolbelt

            -params_file=../../../test/data/cfg/lookup/script_params_big_fs_one.json
